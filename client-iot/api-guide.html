IoT Smart Plant Care — Hướng dẫn Test API

Kiểm thử REST API bằng Postman Web · Base URL: http://localhost:5000

POSTMAN WEB

Bước 1 — Chuẩn bị Postman Web

Truy cập & tạo Workspace

  1. Mở trình duyệt, vào web.postman.co → đăng nhập tài khoản (tạo miễn phí nếu chưa có).
  2. Nhấn Workspaces ở thanh trên → Create Workspace → đặt tên IoT Plant API.
  3. Vào workspace vừa tạo → nhấn New → chọn Collection → đặt tên IoT API Collection.
Postman Web không thể gọi trực tiếp đến localhost do giới hạn trình duyệt. Bạn cần cài Postman Desktop Agent (pop-up sẽ hiện tự động khi gửi request lần đầu) hoặc dùng ứng dụng Postman Desktop.

Tạo Environment (biến môi trường)

  1. Nhấn Environments ở sidebar trái → Create Environment → đặt tên IoT Local.
  2. Thêm biến: base_url = http://localhost:5000 → nhấn Save.
  3. Góc trên phải Postman, chọn environment IoT Local từ dropdown.
  4. Từ nay dùng {{base_url}} thay cho địa chỉ đầy đủ trong mọi request.

GET /api/devices

Các bước thực hiện

  1. Trong collection, nhấn dấu + → chọn Add Request → đặt tên Get Devices.
  2. Chọn method GET. Nhập URL: {{base_url}}/api/devices
  3. Không cần thêm Params hay Body. Nhấn Send.
  4. Kết quả trả về mảng JSON gồm name, current_status, code.

Response mẫu

[
  { "name": "pump",   "current_status": "OFF", "code": "pump"   },
  { "name": "heater", "current_status": "ON",  "code": "heater" },
  { "name": "mist",   "current_status": "OFF", "code": "mist"   }
]
current_status là trạng thái cuối cùng ghi nhận trong action_history. Mặc định "OFF" nếu chưa có lịch sử.

  1. Tạo request mới, đặt tên Sensor History. Method GET, URL {{base_url}}/api/sensor-history.

  3. Nhấn Send. Kiểm tra pagination.totalPages để biết có bao nhiêu trang.

Các bước thực hiện

  1. Tạo request Action History, method GET, URL {{base_url}}/api/action-history.
  2. Tab Params. Ví dụ lọc lệnh BẬT: action = ON
  3. Lọc theo thiết bị: thêm device_id = ID lấy từ kết quả GET /api/devices.
  4. Trường status trong response có thể là: LOADING | SUCCESS | ERROR.

POST /api/control

Request Body (JSON)

FieldKiểuGiá trị hợp lệ
codestringrequired pump | heater | mist | all_lights | blink_mode
actionstringrequired "ON" hoặc "OFF"

Các bước thực hiện

  1. Tạo request Control Device, method POST, URL {{base_url}}/api/control.
  2. Tab Headers → thêm: Content-Type = application/json
  3. Tab Body → chọn raw → dropdown chọn JSON.
  4. Dán body vào ô soạn thảo (ví dụ bật máy bơm): 
    {
  "code": "pump",
  "action": "ON"
}
  5. Nhấn Send. Server trả 200 OK ngay và publish lệnh lên MQTT broker.
  6. ESP32 nhận lệnh → gửi phản hồi device_status_feedback → server cập nhật DB và phát device_update qua Socket.io đến frontend.

Response mẫu

// HTTP 200 — Thành công
{ "success": true, "message": "Command sent to MQTT" }

// HTTP 500 — MQTT lỗi
{ "error": "MQTT client not connected" }
API này cần MQTT broker và ESP32 đang hoạt động để hoàn thành luồng đầy đủ. Khi test chỉ có server, response vẫn là 200 nhưng DB sẽ giữ trạng thái LOADING vĩnh viễn.

Mã thiết bị hợp lệ

codeThiết bịPin ESP32
pumpMáy bơm nướcGPIO 5
heaterMáy sưởiGPIO 3
mistMáy phun sươngGPIO 1
all_lightsToàn bộ đèn—
blink_modeChế độ nhấp nháy đèn—
Dự án dùng Socket.io cho dữ liệu real-time. Postman Web hỗ trợ test Socket.io qua New → Socket.IO với URL http://localhost:5000. Các event cần theo dõi: new_sensor, device_update, init_data.
